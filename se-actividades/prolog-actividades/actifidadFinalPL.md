# actividad

Esta actividad la entregará en equipos de 1 a 3 personas, solo un integrante del equipo subira la actividad, asegurandose de que tenga un archivo `readme.md` en la carpeta raiz del proyecto donde se ponga al inicio la información de: (matriculas, nombres, materia, profesor y descripción del proyecto). No olvide poner un archivo `.gitignore` en su proyecto.

1. para las materias del programa educativo de ISC, programe un sistema experto que apoye al alumno/tutor/gestor en la elcción de cursos respetando su seriación y grado de aprovechamiento del alumno, Por ejmplo:

* no puede llevar matemáticas 2 si no aprobo matemáticas 1, 
* no puede cargar mas de 4 materias si su promedio general fue menos que 80 o si tiene mas de una reprobada.
* debe indicar cuantas veces el alumno ha cursado una materia y las calificaciones en cada curso.
* debe indicar si el alumno debe ser dado de baja (reprobó 3 veces una materia)
* debe encontrar a los alumnos de alto rendimiento (promedio >=90)
* debe mostrar  las materias por semestre y por área
* para abrir un curso para una materia en particular, debe decirme cuantos posibles aspirantes hay

Este sistema debe programarse en swi-prolog creando un servidor con un conjunto de end points definidos.

2. programe un sistema experto usando Tau-prolog que lleve el control de los proyectos de software que se pueden asignar a un programador de acuerdo a su nivel, existe un grupo de programadores con los niveles de (junior, avanzado, senior), y proyectos (A,B,C,D,E,F,G) con diferentes niveles para cada proyecto que pueden ser (bajo, medio, alto, muy alto), cada proyecto requiere asignar personal diferente:
   1. proyecto nivel bajo requiere un desarrollador avanzado y un junior
   2. proyecto nivel medio requiere un desarrollador senior y un avanzado
   3. proyecto nivel alto requiere un desarrollador senior un avanzado y un junior
   4. proyecto nivel muy alto requiere un desarrollador senior 2 avanzados y 2 junior
LLena tu base de conocimientos con 10 desarrolladores ficticios con diferentes niveles, 10 proyectos ficticios con diferentes niveles y deseamos saber lo siguiente:
   1. lista de desarrolladores con sus niveles
   2. lista de proyectos con sus niveles
   3. saber si para un proyecto dado se tiene el personal necesario
   4. saber para un proyecto dado que personal hace falta contratar
3. programa en Go lang un SE para llevar el control de los ingredientes de cada platillo de un menú para indicar que el cocinero pueda seber que ingredientes lleva un guiso y si existen todos los ingredientes para un guiso y que de un alista de ingredientes faltantes paraun  guiso dado.

## valor de la actividad

1 valor 50%
2 valor 40%
3 valor 10%

## avaluación

Se evaluará lo siguiente:
1. que el programa funcione dando los resultado esperados 30%
2. que el código este documentado y sea entendible 10%
3. que use las sentencias enseñadas en clase, en caso de usar otras comentar con el profesor 40%
4. que se entregado en tiempo y forma 20%
5. no se evaluarán proyectos que se entreguen en formato zip o en algun repositorio que nos sea GitHub
